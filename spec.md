# code-validator -- 仕様書

## 概要
AI生成コードのセキュリティ脆弱性・コード品質・依存関係リスクを静的解析する
軽量ツール。Claude, ChatGPT, Copilot等のAIが生成したコードを
マージ前に検証するCI/CDパイプラインの品質ゲートとして設計。

ゼロ外部通信（完全オフライン動作）、最小依存（pydantic のみ）が特徴。

## 機能一覧
- [x] 機能1: セキュリティスキャン（ハードコードされたAPI鍵・パスワード・DB URL検出）
- [x] 機能2: CORS設定不備の検出（ワイルドカードオリジン + credentials）
- [x] 機能3: SQLインジェクションパターン検出
- [x] 機能4: セキュリティヘッダー不足の検出（FastAPIアプリ向け）
- [x] 機能5: コード品質チェック（行長超過・未使用import・関数複雑度）
- [x] 機能6: 依存関係監査（pip-audit / npm audit 委譲）
- [x] 機能7: レポート生成（HTML / JSON / Console）
- [x] 機能8: Git統合（--git-diff で変更ファイルのみスキャン）
- [x] 機能9: カスタム設定（config/validator_config.json）

## 技術スタック
- 言語: Python 3.9+
- 依存: pydantic >= 2.0.0
- オプション: pip-audit >= 2.6.0
- CI: GitHub Actions

## 非機能要件
- 外部API呼び出しなし（完全オフライン動作）
- 終了コード: 0=問題なし、1=Critical/High検出（CI連携用）
- カスタム設定による検出ルールの有効/無効切り替え

## 用語定義
| 用語 | 定義 |
|------|------|
| SEC001-SEC007 | セキュリティ検出ルールID |
| QUAL001 | コード品質検出ルールID |
| DEP001-DEP003 | 依存関係検出ルールID |
| git-diff | HEAD以降の変更ファイルのみスキャンするモード |
